# Acme Dialer — Cold-Call CRM + Browser Dialer

Private, invite-only web app for cold-calling sales leads. The whole point is a
fast calling loop: load a lead → research → call → pitch from a script → take
notes → book a meeting → next lead.

> **Note:** "Acme" is a placeholder brand name — rename it to whatever you like
> (search the repo for `Acme`). The app is fully functional in a local "stub"
> mode with no external accounts; wire up the optional integrations as you go.

## Stack

- **Next.js 16** (App Router, Turbopack)
- **Neon** Postgres via **Drizzle ORM** (`lib/db`)
- **Better Auth** — invite-only, multi-user, admin/worker roles (`lib/auth.ts`)
- **shadcn/ui** + Tailwind v4
- **Telephony**: provider-abstracted (`lib/dialer/`) — stub + Twilio Voice SDK
- **Google Calendar** booking (`lib/google.ts`)

## Features

- Invite-only auth (no public signup); admin creates worker accounts (`/admin`)
- CSV lead import with phone normalization + dedup (`/leads/import`)
- Lead CRM: editable fields, status, callbacks, activity timeline (`/leads`)
- Markdown call scripts, rendered (`/scripts`)
- Campaigns: group leads + attach a script (`/campaigns`)
- **Dial session** (`/dial`): auto-loads next lead, in-app website preview with
  fallback, rendered script, click-to-call, mute/hangup, outcome logging,
  auto-dialer with a configurable timer, in-call notes
- Booking to Google Calendar during a call; bookings + due-callbacks view (`/bookings`)
- Optional AI lead enrichment + call transcription

## Quick start

Requires **Node 20+** and **pnpm** (`npm i -g pnpm`).

```bash
pnpm install
cp .env.example .env     # then fill in DATABASE_URL + BETTER_AUTH_SECRET (see below)
pnpm db:migrate          # create the schema in your database
pnpm seed:admin -- --email you@example.com --name "You" --password "<≥8 chars>"
pnpm dev                 # http://localhost:3000
```

Then sign in at http://localhost:3000/login with the admin you just seeded.

> The app boots and runs with **only** `DATABASE_URL` + `BETTER_AUTH_SECRET` set.
> The dialer runs in simulated mode and the AI/transcription features stay dark
> until you add their keys. See [`.env.example`](.env.example) for every variable.

## Configuration

All configuration is via environment variables in `.env` (gitignored). Copy
[`.env.example`](.env.example) and fill in what you need. The essentials:

| Var | Required? | Purpose |
| --- | --- | --- |
| `DATABASE_URL` / `DATABASE_URL_UNPOOLED` | **Yes** | Postgres (Neon) connection strings |
| `BETTER_AUTH_SECRET` | **Yes** | Auth signing secret — `openssl rand -base64 32` |
| `BETTER_AUTH_URL` | **Yes** | Base URL of the app (`http://localhost:3000` in dev) |
| `DIALER_PROVIDER` | No | `stub` (default) or `twilio` |
| `TWILIO_*` | No | Twilio account / API key / TwiML app / caller ID |
| `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` / `GOOGLE_REDIRECT_URI` | No | Calendar OAuth |
| `ANTHROPIC_API_KEY` / `GOOGLE_PLACES_API_KEY` / `HARVEST_LLM_ENABLED` | No | AI lead enrichment |
| `TRANSCRIPTION_PROVIDER` / `DEEPGRAM_API_KEY` / `TWILIO_RECORD` | No | Call recording + transcription |

### Database

```bash
pnpm db:generate    # create a migration from schema changes
pnpm db:migrate     # apply migrations
pnpm db:studio      # drizzle studio
```

Schema is `lib/db/schema.ts` (app tables) + `lib/db/auth-schema.ts` (generated by
`pnpm auth:generate`).

### First admin

Public signup is disabled (invite-only), so seed the first admin directly:

```bash
pnpm seed:admin -- --email you@example.com --name "You" --password "<≥8 chars>"
```

It's idempotent — re-running promotes an existing user to admin and resets their
password.

## Telephony (Twilio)

The dialer is abstracted behind `lib/dialer/`. It runs on a **simulated** dialer
until Twilio is configured (`DIALER_PROVIDER=twilio` + the `TWILIO_*` vars).
Browser calling uses the Voice SDK; the server mints tokens at
`/api/twilio/token` and serves outbound TwiML at `/api/twilio/voice` (set that
URL as the TwiML App's Voice Request URL).

## Google Calendar

Per-user OAuth (`/api/google/connect` → `/api/google/callback`). Connect from
`/bookings`. Bookings always record locally; they also create a real calendar
event when the user is connected. With a personal-Gmail OAuth client in
External/Testing mode, tokens lapse ~weekly — the bookings page surfaces a
"Reconnect" prompt.

## Deploy

The app is a standard Next.js project and deploys to any Next-compatible host
(Vercel, etc.). Set the same environment variables in your host's dashboard and
run `pnpm db:migrate` after schema changes before deploying.

## Before dialing real numbers (compliance)

This is outbound cold-calling. Before live dials: scrub against the National
Do-Not-Call Registry, honor TCPA calling-hour/opt-out rules, and register the
caller ID for STIR/SHAKEN attestation. None of this is built into the app —
handle it before flipping `DIALER_PROVIDER=twilio`.
